# 变更提案: src 函数注释补充

## 需求背景
src 目录内存在较多函数缺少清晰的功能说明，阅读成本高，维护和回归时容易误解。
本次变更仅补充必要注释，明确函数/方法的作用与边界，不改变任何逻辑行为。

## 变更内容
1. 为 `context_retrieval`、`command_parser`、`main` 中缺少说明的函数补充简洁注释。
2. 统一注释风格为“说明函数做什么、输入输出与限制”，避免重复代码细节。

## 影响范围
- **模块:** context_retrieval, command_parser, main
- **文件:** src/context_retrieval/*.py, src/command_parser/*.py, src/main.py
- **API:** 无
- **数据:** 无

## 核心场景

### 需求: 上下文检索函数注释补充
**模块:** context_retrieval
为检索流水线、检索策略与评分逻辑的关键函数补充注释，方便维护人员快速定位职责。

#### 场景: 维护人员快速理解检索流程
维护人员阅读 `context_retrieval` 代码时，能通过函数注释快速了解其用途与边界。
- 检索流程与降级策略的意图清晰
- 关键选择逻辑有明确说明

### 需求: 指令解析与入口函数注释补充
**模块:** command_parser, main
为解析流程和入口调用路径补充注释，帮助维护人员理解系统整体入口与解析边界。

#### 场景: 维护人员理解解析与入口逻辑
维护人员阅读解析与入口代码时，能通过注释理解解析职责与入口流程。
- 解析逻辑的输入输出清晰
- 入口函数调用链明确

## 风险评估
- **风险:** 注释过多或描述不准确可能误导阅读者
- **缓解:** 仅补充必要注释，保持简洁并与代码行为一致
